# 推波助澜择时策略

## 1. 项目简介

本项目实现了一个基于市场广度指标的 A 股指数择时策略。核心思想是使用全市场股票中“近似涨停”和“近似跌停”数量的差值，构造“推波助澜比率”，再对该比率计算 Hull Moving Average（HMA），据此生成择时信号，并以指数收益作为策略收益载体进行回测。

当前代码会自动完成以下工作：

- 下载或读取本地缓存的 A 股日频数据
- 计算推波助澜比率
- 计算短期和长期 HMA
- 生成买卖信号和仓位序列
- 进行指数择时回测
- 导出 Excel 报表、HTML 报告和涨停/跌停股票列表

## 2. 策略定位

- 策略名称：推波助澜择时策略
- 策略类型：市场广度因子驱动的指数择时策略
- 资产类别：A 股股票市场信号 + A 股指数收益承载
- 交易对象：指数收益序列（当前基准为上证指数 `000001.SH`）
- 数据频率：日频
- 信号频率：日频
- 持仓方式：0/1 仓位，空仓或满仓
- 交易方向：仅做多，不做空

## 3. 核心逻辑

策略先在全市场非 ST 股票中，统计每个交易日：

- 涨幅大于 `+9.5%` 的股票数量
- 跌幅小于 `-9.5%` 的股票数量

然后构造：

- 涨停比率 = 涨停股票数 / 当日有效股票总数
- 跌停比率 = 跌停股票数 / 当日有效股票总数
- 推波助澜比率 = 涨停比率 - 跌停比率

在此基础上，进一步计算：

- 短期 HMA：30 日
- 长期 HMA：100 日
- HMA 比率：`HMA30 / HMA100`

### 买入条件

同时满足以下条件时，策略进入满仓：

- `HMA30 / HMA100 > 1.15`
- `HMA30 > 0`
- `HMA100 > 0`

### 卖出条件

只要满足以下任一条件，策略转为空仓：

- `HMA30 / HMA100 <= 1.15`
- `HMA30 <= 0`
- `HMA100 <= 0`

## 4. 回测设定

- 起始日期：`2024-01-01`
- 结束日期：运行日的前一日
- 基准指数：上证指数 `000001.SH`
- 初始资金：`1,000,000`
- 单边交易成本：`0.1%`
- 无风险利率：`2%`

说明：

- 策略信号由全市场股票广度数据产生
- 策略收益目前使用指数收益率序列进行回测
- 这意味着它是“用股票市场内部结构做信号，用指数做执行载体”的日频择时模型

## 5. 数据来源与缓存

数据来源为 Tushare，主要包括：

- 股票基础信息
- 全市场股票日线行情
- 指数日线行情

本项目使用本地 Parquet 做缓存，主要目录包括：

- `data_parquet/`：股票日线、指数数据、推波助澜指标缓存
- `stock_lists/`：每日涨停/跌停股票列表
- `output/`：策略 Excel 和 HTML 报告输出

## 6. 输出结果

策略运行后会输出：

- `push_wave_daily_positions.xlsx`
  - 每日持仓
  - 交易记录
  - 技术指标
  - 涨停股票列表
  - 跌停股票列表
  - 绩效汇总
- `strategy_dashboard.html`
  - 可视化策略报告
  - 近期和完整净值曲线
  - 绩效指标摘要

### 当前版本特点

### 尚未完全完善的部分

- 交易日历目前仍偏向通用工作日，不是严格的交易所日历
- 股票池更新逻辑仍偏静态，存在进一步优化空间
- GitHub Actions 部署前仍需确保 Secrets 和推送权限配置正确
- 回测与执行层目前是指数级别，不是个股组合实盘执行框架

## 9. 适用场景

本策略更适合以下用途：

- 作为 A 股市场情绪/广度的日频择时观察工具
- 作为指数增强或指数风控的上层择时模块
- 作为后续扩展成多指数轮动或 ETF 执行策略的信号层

## 11. 风险与限制

- 本策略属于历史回测模型，不代表未来收益
- A 股涨跌停制度、ST 规则、停复牌处理等现实约束，会影响信号解释
- 当前股票池过滤逻辑仍可能引入一定偏差
- 交易成本、滑点、指数跟踪误差在实盘中可能高于回测假设
